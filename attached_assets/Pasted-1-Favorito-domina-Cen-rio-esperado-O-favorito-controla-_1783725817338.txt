Uma recomendação de arquitetura

Em vez de cadastrar essas combinações manualmente, modele cada uma como um template. Exemplo:

{
  "id": "favorite_open_game",
  "title": "Favorito em jogo aberto",
  "legs": [
    "HOME_WIN",
    "OVER_2_5",
    "BTTS_YES"
  ],
  "conditions": {
    "home_win_probability": ">=0.55",
    "expected_goals": ">=2.6"
  }
}

E outro:

{
  "id": "slow_first_half",
  "title": "Primeiro tempo equilibrado",
  "legs": [
    "HT_DRAW",
    "FT_HOME_WIN",
    "OVER_2_5"
  ],
  "conditions": {
    "first_half_goals": "<=1.0",
    "home_win_probability": ">=0.50"
  }
}

Com essa abordagem, você consegue ter uma biblioteca de dezenas ou centenas de combinações, e o sistema publica apenas as que são compatíveis com as probabilidades e regras de cada partida. Isso escala muito melhor do que criar mercados específicos jogo a jogo.
