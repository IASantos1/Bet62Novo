Para Tênis, a melhor prática é criar um Settlement Engine separado, porque a lógica é baseada em sets e games, não em gols.

A estrutura pode ser assim:

Tennis Settlement Engine
│
├── Match Winner
├── Set Winner
├── Exact Sets
├── Set Handicap
├── Game Handicap
├── Total Games
├── Set Total Games
├── Correct Score
├── Tie-break
├── Specials
└── Combo Settlement

Estrutura recomendada
Tennis Settlement

├── Match
│   ├── Match Winner
│   ├── Exact Sets
│   └── Match Handicap
│
├── Sets
│   ├── Set Winner
│   ├── Correct Set Score
│   ├── Set Handicap
│   ├── Set Totals
│   └── Tie-break
│
├── Games
│   ├── Total Games
│   ├── Game Handicap
│   ├── Player Games
│   └── Odd/Even Games
│
├── Specials
│   ├── First Break
│   ├── Number of Breaks
│   ├── Tie-break Yes/No
│   └── Race to X Games
│
└── Combo Settlement
Fluxo do Settlement
Feed Oficial do Tênis
        │
        ▼
Resultado Final
        │
        ▼
Liquida todos os mercados individuais
        │
        ├── Match Winner
        ├── Set Winner
        ├── Exact Sets
        ├── Correct Score
        ├── Handicap Sets
        ├── Handicap Games
        ├── Total Games
        ├── Set Totals
        └── Especiais
        │
        ▼
Grava WIN / LOSE / VOID / HALF_WIN / HALF_LOSE
        │
        ▼
Liquida os bilhetes e combinações

Essa arquitetura cobre os mercados que você mostrou e também facilita adicionar novos mercados de tênis (como tie-break, break point, race to games e especiais) sem alterar o núcleo do Settlement Engine.
