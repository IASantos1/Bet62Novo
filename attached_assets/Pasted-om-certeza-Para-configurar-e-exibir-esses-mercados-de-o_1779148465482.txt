Passos para Configurar e Exibir os Mercados
1. Fazer a Requisição à API
Primeiro, você precisa obter os dados da partida e das odds.

Exemplo em JavaScript (usando fetch):


const API_KEY = "YOUR_API_KEY"; // Substitua pela sua chave real
const MATCH_ID = 4609057; // Exemplo de Match ID
const SPORT = "football"; // Ou basketball, tennis, etc.

async function fetchMatchOdds(matchId, sport) {
  try {
    const response = await fetch(
      `https://v2.${sport}.sportsapipro.com/api/match/${matchId}/odds/all`,
      {
        headers: {
          "x-api-key": API_KEY,
        },
      }
    );

    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    const data = await response.json();
    return data;
  } catch (error) {
    console.error("Erro ao buscar odds:", error);
    return null;
  }
}

// Exemplo de uso:
fetchMatchOdds(MATCH_ID, SPORT).then((oddsData) => {
  if (oddsData) {
    console.log("Dados de Odds:", oddsData);
    displayOdds(oddsData); // Chame uma função para exibir os dados
  }
});
2. Processar e Organizar os Dados
A resposta da API pode ter múltiplos bookmakers e cada um com seus markets. Você provavelmente vai querer:

Escolher um bookmaker preferencial (ex: o primeiro da lista, ou um específico pelo id).
Iterar pelos markets desse bookmaker.

function displayOdds(oddsData) {
  const preferredBookmaker = oddsData.bookmakers[0]; // Pegar o primeiro bookmaker como exemplo
  if (!preferredBookmaker) {
    console.log("Nenhum bookmaker encontrado.");
    return;
  }

  const oddsContainer = document.getElementById("odds-container"); // Um elemento HTML onde você exibirá as odds
  oddsContainer.innerHTML = `<h2>Odds de ${preferredBookmaker.name}</h2>`;

  preferredBookmaker.markets.forEach((market) => {
    const marketDiv = document.createElement("div");
    marketDiv.className = "market";
    marketDiv.innerHTML = `<h3>${market.name}</h3>`;

    const outcomesList = document.createElement("ul");
    market.outcomes.forEach((outcome) => {
      const outcomeItem = document.createElement("li");
      outcomeItem.className = `outcome ${outcome.status}`; // Adicionar classe para status (ativo/inativo)
      outcomeItem.innerHTML = `
        <span>${outcome.name}</span>
        <span class="odds-value">${outcome.odds.toFixed(2)}</span>
      `;
      outcomesList.appendChild(outcomeItem);
    });

    marketDiv.appendChild(outcomesList);
    oddsContainer.appendChild(marketDiv);
  });
}
3. Mapear e Exibir Mercados Específicos
Para os mercados que você listou (1X2, Over/Under, BTTS, etc.), você pode usar o market.name ou market.id para identificá-los e exibi-los de forma específica.

Exemplo de como lidar com mercados específicos:


function displaySpecificOdds(oddsData) {
  const preferredBookmaker = oddsData.bookmakers[0];
  if (!preferredBookmaker) return;

  const markets = preferredBookmaker.markets;

  // --- 1X2 (Match Winner) ---
  const matchWinnerMarket = markets.find(
    (m) => m.name === "Match Winner" || m.name === "1X2"
  );
  if (matchWinnerMarket) {
    const homeOdds = matchWinnerMarket.outcomes.find((o) => o.name === "Home")?.odds;
    const drawOdds = matchWinnerMarket.outcomes.find((o) => o.name === "Draw")?.odds;
    const awayOdds = matchWinnerMarket.outcomes.find((o) => o.name === "Away")?.odds;
    console.log(`1X2: Casa ${homeOdds || '-'} | Empate ${drawOdds || '-'} | Fora ${awayOdds || '-'}`);
    // Renderize isso em seu HTML
  }

  // --- Over/Under 2.5 Gols ---
  const totalGoalsMarket = markets.find(
    (m) => m.name === "Total Goals Over/Under"
  );
  if (totalGoalsMarket) {
    const over2_5 = totalGoalsMarket.outcomes.find((o) => o.name === "Over 2.5")?.odds;
    const under2_5 = totalGoalsMarket.outcomes.find((o) => o.name === "Under 2.5")?.odds;
    console.log(`Over/Under 2.5: Over ${over2_5 || '-'} | Under ${under2_5 || '-'}`);
    // Renderize isso em seu HTML
  }

  // --- Ambas as Equipes Marcam (BTTS) ---
  const bttsMarket = markets.find(
    (m) => m.name === "Both Teams to Score" || m.name === "BTTS"
  );
  if (bttsMarket) {
    const yesOdds = bttsMarket.outcomes.find((o) => o.name === "Yes")?.odds;
    const noOdds = bttsMarket.outcomes.find((o) => o.name === "No")?.odds;
    console.log(`BTTS: Sim ${yesOdds || '-'} | Não ${noOdds || '-'}`);
    // Renderize isso em seu HTML
  }

  // ... e assim por diante para cada tipo de mercado que você deseja exibir.
  // Você pode criar um mapa de nomes de mercado para funções de renderização.
}

4. Considerações de Design e UX
Escolha do Bookmaker: Decida se você quer exibir odds de um único bookmaker, comparar odds de vários, ou talvez exibir a "melhor odd" para cada resultado.
Agrupamento: Agrupe mercados relacionados (ex: todos os mercados de gols, todos os mercados de handicap).
Atualização de Odds: As odds mudam rapidamente, especialmente em jogos ao vivo. Você precisará de um mecanismo para atualizar as odds periodicamente (polling com lastUpdateId ou WebSockets para dados ao vivo).
Status das Odds: Preste atenção ao status de cada outcome. Odds podem ser active, suspended, settled, etc. Exiba apenas as ativas ou indique claramente o status.
Internacionalização: Os name dos mercados e outcomes são em inglês. Se sua aplicação for para outro idioma, você precisará mapear esses nomes para traduções.
