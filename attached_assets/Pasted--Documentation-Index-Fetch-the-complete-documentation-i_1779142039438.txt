## Your First API Call

All V2 endpoints use the pattern `https://v2.{sport}.sportsapipro.com/api/{endpoint}`. Authentication is via the `x-api-key` header.

### Get Live Football Scores

<CodeGroup>
  ```bash cURL theme={null}
  curl -H "x-api-key: YOUR_API_KEY" \
    "https://v2.football.sportsapipro.com/api/live"
  ```

  ```javascript JavaScript theme={null}
  const response = await fetch("https://v2.football.sportsapipro.com/api/live", {
    headers: { "x-api-key": "YOUR_API_KEY" }
  });
  const data = await response.json();
  console.log(`${data.count} live matches`);
  data.events.forEach(e => {
    console.log(`${e.homeTeam} ${e.homeScore} - ${e.awayScore} ${e.awayTeam} (${e.status})`);
  });
  ```

  ```python Python theme={null}
  import requests

  response = requests.get(
      "https://v2.football.sportsapipro.com/api/live",
      headers={"x-api-key": "YOUR_API_KEY"}
  )
  data = response.json()
  for event in data.get("events", []):
      print(f"{event['homeTeam']} {event['homeScore']} - {event['awayScore']} {event['awayTeam']} ({event['status']})")
  ```
</CodeGroup>

### Response (Football `/api/live`)

```json theme={null}
{
  "count": 58,
  "events": [
    {
      "id": 15269945,
      "homeTeam": "Charlotte FC",
      "awayTeam": "Nashville SC",
      "homeScore": 1,
      "awayScore": 2,
      "status": "2nd half",
      "tournament": "MLS",
      "slug": "charlotte-fc-nashville-sc",
      "startTimestamp": 1775950200
    }
  ]
}
```

<Note>
  Football's `/api/live` returns a **simplified** format with flat score values. Other sports (Basketball, Baseball, Tennis) return **enhanced** formats with score breakdowns — see examples below.
</Note>

## Common V2 Endpoint Patterns

All V2 sports share the same endpoint structure:

| Endpoint                           | Description                         |
| ---------------------------------- | ----------------------------------- |
| `GET /api/live`                    | Currently live matches (simplified) |
| `GET /api/today`                   | Today's full schedule (detailed)    |
| `GET /api/schedule/{date}`         | Schedule by date (YYYY-MM-DD)       |
| `GET /api/search?q={query}`        | Search teams, players, tournaments  |
| `GET /api/countries`               | Country/category list               |
| `GET /api/match/{id}`              | Full match details                  |
| `GET /api/match/{id}/statistics`   | Match statistics                    |
| `GET /api/match/{id}/lineups`      | Match lineups                       |
| `GET /api/match/{id}/incidents`    | Match events/incidents              |
| `GET /api/match/{id}/odds`         | Match betting odds                  |
| `GET /api/tournament/{id}/info`    | Tournament details                  |
| `GET /api/tournament/{id}/seasons` | Tournament seasons                  |
| `GET /api/teams/{id}`              | Team details                        |
| `GET /api/players/{id}`            | Player details                      |
| `GET /status`                      | Account status & usage              |

<Warning>
  V2 endpoints use `/api/live`, **not** `/api/football/live`. The sport is determined by the subdomain (`v2.football.sportsapipro.com`), not the URL path.
</Warning>
