# Basketball V2 API Overview

> Enhanced Basketball API with 115 endpoints: box scores, player stats, playoff brackets, conference standings, and more

## Base URL

```
https://v2.basketball.sportsapipro.com
```

## What's Different in V2?

The V2 Basketball API provides **enhanced data coverage** with significantly more endpoints and richer data compared to V1:

## Authentication

Same `x-api-key` header as V1 — your existing API key works for both versions:

```bash theme={null}
curl -X GET "https://v2.basketball.sportsapipro.com/api/live" \
  -H "x-api-key: YOUR_API_KEY"
```

## Important: Entity ID Discovery

<Warning>
  Entity IDs (players, matches, teams, venues, referees) are **numeric** and must be discovered dynamically — never hardcode them.

  **Discovery flow:**

  1. Use `/api/search?q=lakers` to find team/player IDs
  2. Use `/api/live` or `/api/schedule/{date}` to find match IDs
  3. Use `/api/match/{id}` to extract venue, referee, and player IDs from the response
  4. Use `/api/tournament/{id}/seasons` to discover valid season IDs
</Warning>

## Key Differences from Football

<Note>
  **Scoring:** Basketball uses `period1`, `period2`, `period3`, `period4`, `overtime`, `normaltime`, and `current` fields in the score object — unlike football which uses halftime periods.

  **Lineups:** Starting 5 + bench (not 11 as in football).

  **Incidents:** Include baskets, fouls, free throws, and timeouts (not goals/cards).

  **Standings:** Typically have conference groups (Eastern/Western for NBA) instead of a single table.

  **No shotmap or heatmap endpoints** — these are football-specific features.

  **Positions:** G (Guard), F (Forward), C (Center) instead of football positions.
</Note>

## Example Requests

<CodeGroup>
  ```bash cURL theme={null}
  curl -X GET "https://v2.basketball.sportsapipro.com/api/live" \
    -H "x-api-key: YOUR_API_KEY"
  ```

  ```javascript JavaScript theme={null}
  const response = await fetch(
    'https://v2.basketball.sportsapipro.com/api/live',
    { headers: { 'x-api-key': 'YOUR_API_KEY' } }
  );
  const data = await response.json();
  // { success: true, count: 48, events: [...] }
  ```

  ```python Python theme={null}
  import requests

  response = requests.get(
      'https://v2.basketball.sportsapipro.com/api/live',
      headers={'x-api-key': 'YOUR_API_KEY'}
  )
  data = response.json()
  ```
</CodeGroup>

## UI Design Ideas

* **Score display**: Show Q1-Q4 with totals, highlight current quarter during live games
* **Box scores**: Full player stat lines (PTS, REB, AST, STL, BLK, FG%, 3P%, FT%)
* **Standings**: Conference tables (Eastern/Western), show W-L record, PCT, GB
* **Playoff bracket**: Show 4 rounds (First Round → Conf Semis → Conf Finals → Finals)
* **Player profile**: Position (G/F/C), height, weight, jersey number, draft info
