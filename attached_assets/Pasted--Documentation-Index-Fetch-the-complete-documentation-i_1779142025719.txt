## Three API Versions

SportsAPI Pro offers three API versions, each optimized for different use cases:

| Version | Sports    | Best For                     | Latency   | WebSocket |
| ------- | --------- | ---------------------------- | --------- | --------- |
| **V1**  | 9 sports  | Speed-critical score updates | 1–2s      | ✅ Yes     |
| **V2**  | 25 sports | Deep data, rich analytics    | 3–5s      | ✅ Yes     |
| **V3**  | 34 sports | Widest coverage, checksums   | 15s cache | ❌ No      |

<Tip>
  **V2 is recommended for most use cases.** It provides the richest data with 100+ endpoints per sport. Use V1 when you need the fastest possible score updates, and V3 when you need niche sports not available in V2.
</Tip>

## Quick Example

<CodeGroup>
  ```bash cURL theme={null}
  curl -H "x-api-key: YOUR_API_KEY" \
    "https://v2.football.sportsapipro.com/api/live"
  ```

  ```javascript JavaScript theme={null}
  const response = await fetch("https://v2.football.sportsapipro.com/api/live", {
    headers: { "x-api-key": "YOUR_API_KEY" }
  });
  const data = await response.json();
  // { count: 58, events: [{ id, homeTeam, awayTeam, homeScore, awayScore, status, tournament }] }
  ```

  ```python Python theme={null}
  import requests

  response = requests.get(
      "https://v2.football.sportsapipro.com/api/live",
      headers={"x-api-key": "YOUR_API_KEY"}
  )
  data = response.json()
  # { "count": 58, "events": [{ "id", "homeTeam", "awayTeam", ... }] }
  ```
</CodeGroup>
